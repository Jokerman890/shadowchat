# ADR-0006 Bridges Security Model

## Entscheidung
Externe oder gebridgte Kontexte erhalten ein eigenes Trust- und Security-Modell.

## Begründung
- native und externe Kommunikation sind nicht identisch
- Nutzer müssen den Unterschied sofort erkennen

## Konsequenzen
- eigener visueller Status für externe Quellen
- Security Center zeigt getrennte Trust-Einstufungen
- Bridge-Funktionalität ist nicht Teil des MVP-Kerns
